ZRawMasterFile
--------------

- Record storage format

    ZRawMasterFile records are composed of URF formatted data.

- Index keys

Index key is no more of fixed length size.

Index keys allow varying size objects.
Index key stores URF formatted data and not Natural formatted data


Search engines comparizon routines are modified accordingly.

All key fields must be present : return Error ZS_OMITTED if key field is not present






Indexes :

    - ZRawMasterFile :
        . low level : only write ZDataBuffer to index ZMasterFile.
                    ZDataBuffer is set with key fields values in URF format.

        . level + 1 : extract URF fields from ZRawMasterFile record (given as a ZDataBuffer) using their position
                NB: fields are stored within record in URF format

                test if field exists using ZFieldPresence.
                if not exits : return error ZS_OMITTED
                if exists extract and format key ZDataBuffer
    
    - ZMasterFile :
        . level + 2 : extract field using dictionary rank following level + 1 processing (field name is ignored)
        
ZRawIndexControlBlock :

    Key universal size is no more stored (it means nothing in a varying field context)
    
    key field components are stored as their positions within record.
